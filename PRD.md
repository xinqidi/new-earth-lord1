# 产品需求文档 (PRD)
# 《地球新主 Earth Lord》

**版本:** 1.0
**创建日期:** 2025-12-23
**产品类型:** LBS 生存类游戏
**目标平台:** iOS (全球175个国家和地区)

---

## 1. 产品概述

### 1.1 产品定位
《地球新主》是一款基于真实地理位置的末日生存手游，融合了 GPS 圈地、探索搜刮、建造经营、社交交易等多种玩法。玩家通过现实世界中的行走来圈占领地、探索地标、收集资源，在废土世界中重建文明。

### 1.2 核心价值主张
- **Walk to Earn**: 将现实世界的行走转化为游戏内资源和成就
- **真实地理映射**: 利用真实地标和地理数据构建游戏世界
- **社交协作**: 基于地理位置的玩家互动和资源交易
- **希望朋克美学**: 在末日废土背景下传递重建与希望的主题

### 1.3 目标用户画像
- **主要用户**: 18-35岁，喜欢运动健身、户外探索的年轻人
- **次要用户**: 游戏爱好者、LBS游戏玩家（宝可梦GO用户）
- **地域**: 初期聚焦亚洲和北美市场，后续拓展全球

### 1.4 市场对标
- **宝可梦GO**: 累计收入超过60亿美元，证明LBS游戏的商业价值
- **Ingress**: 地理位置游戏的先驱
- **Pikmin Bloom**: 任天堂的步行激励游戏

## 2. 产品背景与世界观

### 2.1 故事背景
- **2048年**: "终末战争"爆发，全球文明在72小时内崩塌，99%的人口消失
- **2051年**: 幸存者从废墟中走出，《新开拓者协议》颁布
- **核心法则**: "凡以双足丈量之无主土地，皆为开拓者所有"
- **视觉风格**: "希望朋克" - 废土之上的新生，绝望中的希望

### 2.2 游戏核心循环
```
登录注册 → 圈地占领 → 探索搜刮 → 获得资源
   ↑                              ↓
   │         资源用途：
   │         • 建造建筑（消耗资源）
   │         • 交易换物（流通资源）
   │         • 生存消耗（食物/水）
   │                              ↓
   └──────── 通讯社交 ← 成就排行 ← 发展壮大
```

## 3. 功能需求规格

### 3.1 第一部分：登录与圈地系统

#### 3.1.1 用户登录注册系统 (P0 - 必须有)
**功能描述**:
用户通过多种方式注册和登录游戏，建立游戏身份。

**需求详情**:
- 支持三种登录方式:
  - 邮箱注册（用户名+密码）
  - Apple 登录（Sign in with Apple）
  - Google 登录（OAuth 2.0）
- 登录成功后进入游戏主界面
- 会话管理：支持多设备登录，自动刷新Token
- 密码找回：邮箱验证码重置密码

**技术实现**:
- 使用 Supabase Auth 作为认证后端
- 前端使用 SwiftUI 构建登录界面
- 支持自动登录和Remember Me功能

**验收标准**:
- [ ] 三种登录方式均可正常使用
- [ ] 登录失败有明确错误提示
- [ ] 会话过期自动跳转登录页
- [ ] 密码重置流程完整可用

---

#### 3.1.2 GPS圈地系统 (P0 - 核心玩法)
**功能描述**:
玩家通过实际行走，用GPS轨迹在地图上圈出多边形区域，占领领地。

**需求详情**:
- **开始圈地**:
  - 点击"开始开拓"按钮，开始记录GPS轨迹
  - 实时在卫星地图上绘制蓝色轨迹线
  - 显示当前行走距离和时间

- **轨迹闭合**:
  - 当玩家回到起点附近（50米内）时，自动闭合为多边形
  - 闭合后轨迹变为绿色
  - 自动计算领地面积（平方米）

- **合法性验证**:
  - 检查多边形是否自交（不能有交叉）
  - 检查是否与其他玩家领地重叠
  - 检查面积是否满足最小要求（500m²）
  - 验证通过后领地永久归属玩家

- **地图显示**:
  - 使用苹果原生 MapKit 显示卫星地图
  - 已占领的领地以半透明绿色显示
  - 其他玩家的领地以灰色显示
  - 支持地图缩放和拖动

**技术实现**:
- 使用 CoreLocation 获取GPS定位
- 使用 MapKit 绘制地图和多边形
- 使用 PostGIS（Supabase内置）存储地理数据
- 使用地理空间查询判断领地重叠

**验收标准**:
- [ ] GPS定位精度在10米以内
- [ ] 轨迹绘制流畅无卡顿（60fps）
- [ ] 面积计算误差在5%以内
- [ ] 重叠检测准确率100%
- [ ] 支持离线记录轨迹，联网后同步

#### 3.1.3 领地管理系统 (P1 - 重要)
**功能描述**:
玩家可以管理自己占领的领地，包括命名、查看信息、开启交易等。

**需求详情**:

**验收标准**:

- [ ] 回收机制按时触发

### 3.2 第二部分：探索与资源获取

#### 3.2.1 POI发现与搜刮系统 (P0 - 核心玩法)
**功能描述**:
玩家在现实世界中行走，当接近真实地标（POI）时，可以进行搜刮获得资源。

**需求详情**:
- **POI自动发现**:
  - 使用地理围栏技术，玩家进入POI 100米范围内自动触发
  - 弹出"发现兴趣点"提示框
  - 显示POI名称、类型、图标

- **POI类型与资源映射**:
  - 医院/药店 → 药品（急救包、抗生素）
  - 超市/餐厅 → 食物（罐头、饮用水）
  - 工厂/工地 → 金属材料
  - 公园/林地 → 木材
  - 银行/珠宝店 → 稀有材料

- **搜刮机制**:
  - 点击"搜刮"按钮开始搜刮
  - 显示探索成就页面：距离、时长
  - 随机生成3-5种物品，数量根据POI等级决定
  - 同一POI每日只能搜刮一次
  - 搜刮成功后物品自动进入背包

- **探索成就**:
  - 记录探索距离（累计）
  - 记录探索时长（累计）
  - 记录探索POI数量

**技术实现**:
- 使用 MapKit Local Search 获取真实POI数据
- 使用 CLRegion 实现地理围栏
- 搜刮逻辑使用随机算法，根据POI类型和稀有度生成物品

**验收标准**:
- [ ] POI识别准确率95%以上
- [ ] 地理围栏触发延迟在5秒内
- [ ] 资源生成符合概率分布
- [ ] 每日限制机制正常工作

### 3.3 第三部分：建造与交易系统

#### 3.3.1 建造系统 (P0 - 核心玩法)
**功能描述**:
玩家使用收集到的资源，在领地上建造各类建筑。

**需求详情**:
- **建筑分类**:
  - 生存类：篝火（取暖）、庇护所（休息）
  - 储存类：小仓库、中仓库、大仓库
  - 生产类：农田（生产食物）、工作台（加工材料）
  - 能源类：太阳能板、风力发电机

- **建筑等级**:
  - Tier 1：生存基础（木材+石头）
  - Tier 2：功能扩展（金属+混凝土）
  - Tier 3：高级设施（稀有材料+电子元件）

- **建造流程**:
  1. 进入"建造"页面，浏览建筑模板
  2. 选择要建造的建筑，查看所需资源
  3. 检查背包资源是否足够
  4. 点击"开始建造"，在领地地图上拖动选择位置
  5. 确认位置后扣除资源，建筑进入"建造中"状态
  6. 建造完成后变为"待激活"状态

- **资源不足处理**:
  - 显示"资源不足"提示
  - 显示缺少的资源和数量
  - 提供"前往探索"和"前往交易"入口

**技术实现**:
- 建筑模板存储在 Supabase
- 使用状态机管理建筑状态
- 资源扣除使用事务保证原子性

**验收标准**:
- [ ] 建筑模板完整（15种建筑）
- [ ] 资源检查逻辑正确
- [ ] 建筑位置可自由拖动
- [ ] 资源扣除不可回退

---

#### 3.3.2 建筑状态管理 (P1 - 重要)
**功能描述**:
建筑有多种状态，不同状态支持不同的操作。

**需求详情**:
- **建筑状态**:
  - 建造中：显示剩余时间
  - 待激活：需要玩家手动激活
  - 运行中：正常工作，可升级
  - 损坏：需要维修
  - 已拆除：不可恢复

- **状态流转规则**:
  - 建造中 → 待激活（时间到达）
  - 待激活 → 运行中（玩家激活）
  - 运行中 → 损坏（随机事件或时间衰减）
  - 损坏 → 运行中（玩家维修）
  - 任何状态 → 已拆除（玩家拆除）

- **状态限制**:
  - 只能升级"运行中"状态的建筑
  - 只能维修"损坏"状态的建筑
  - "建造中"状态不可操作

**验收标准**:
- [ ] 状态流转符合规则
- [ ] 非法操作有明确提示
- [ ] 状态变化实时更新UI

#### 3.3.3 交易系统 (P1 - 重要)
**功能描述**:
玩家之间可以发布物物交易挂单，以物易物获取所需资源。

**需求详情**:
- **发布交易**:
  - 点击"+"发布新交易
  - 选择"我提供"的物品（从背包选择）
  - 选择"我需要"的物品（从物品库选择）
  - 设置交易地点（当前领地）
  - 发布挂单

- **交易范围**:
  - 只有进入发布者领地100米范围内的玩家可见该挂单
  - 需要发布者开启"允许交易"设置

- **交易流程**:
  1. 买家浏览附近的交易挂单
  2. 选择感兴趣的交易，点击"接受交易"
  3. 系统检查双方资源是否足够
  4. 检查通过后扣除双方物品，完成交换
  5. 双方收到交易成功通知

- **交易记录**:
  - 记录交易时间、对象、物品
  - 可查看历史交易记录

**技术实现**:
- 使用地理空间查询筛选附近的挂单
- 交易使用数据库事务保证原子性
- 实时推送交易通知

**验收标准**:
- [ ] 交易范围限制准确
- [ ] 交易过程无法作弊
- [ ] 交易记录完整可查

  - 生命值降到0后复活
  - 损失50%背包物品
  - 损失10%金币

**技术实现**:
- 使用定时器每10分钟检查一次

**验收标准**:
- [ ] 数值衰减准确

## 4. 非功能需求

### 4.1 性能需求
- **启动时间**: 冷启动 < 3秒，热启动 < 1秒
- **GPS定位**: 精度10米以内，刷新频率1次/秒
- **地图渲染**: 60fps流畅显示
- **消息延迟**: 实时消息延迟 < 2秒
- **数据同步**: 后台同步不影响前台使用

### 4.2 安全需求
- **数据加密**: 所有网络传输使用HTTPS
- **身份认证**: 使用JWT Token，有效期7天
- **防作弊**: GPS轨迹验证速度，超过20km/h视为作弊
- **隐私保护**: 用户位置数据脱敏处理，不暴露精确坐标

### 4.3 兼容性需求
- **iOS版本**: 支持 iOS 16.0 及以上
- **设备**: 支持 iPhone 和 iPad
- **网络**: 支持 WiFi 和蜂窝数据，支持离线模式

### 4.4 可用性需求
- **可用性目标**: 99.9%（每月停机时间 < 43分钟）
- **数据备份**: 每日自动备份，保留30天
- **灾难恢复**: RTO < 4小时，RPO < 1小时

## 8. 风险与应对

### 8.1 技术风险
- **GPS精度问题**:
  - 风险：城市高楼、室内定位不准
  - 应对：使用融合定位算法，提示用户到开阔地带

- **地图数据限制**:
  - 风险：某些地区POI数据不完整
  - 应对：允许玩家手动标记POI

### 8.2 运营风险
- **用户作弊**:
  - 风险：使用虚拟定位软件
  - 应对：GPS轨迹速度验证、行为分析封号

- **初期用户少**:
  - 风险：附近没有其他玩家，社交功能形同虚设
  - 应对：加强单机玩法（探索、建造），降低社交依赖

### 8.3 商业风险
- **付费转化率低**:
  - 风险：用户不愿意付费
  - 应对：优化付费点设计，提供高性价比内容

- **竞品威胁**:
  - 风险：宝可梦GO等大厂产品竞争
  - 应对：差异化定位（生存+建造），聚焦细分市场
